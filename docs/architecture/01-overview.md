# SentinelHub 整体架构概览

## 1. 项目定位

SentinelHub 是企业级安全办公平台，面向中大型组织的终端安全、数据防泄漏、网络准入与零信任访问场景。平台以 **统一PC 安全客户端** 为数据采集与策略执行入口，以 **策略引擎** 为决策中枢，以 **审计日志** 为横切能力，支撑合规与取证。

## 2. 设计原则

| 原则 | 说明 |
|------|------|
| **模块解耦** | 业务能力按域拆分微服务，通过事件总线与 API 协作，避免单体耦合 |
| **策略统一** | 各管控能力共享策略模型、设备上下文与执行通道，减少 Agent 冗余逻辑 |
| **安全默认** | 传输加密、mTLS、最小权限、敏感数据脱敏，审计不可篡改 |
| **可扩展** | 插件化检测规则、开放 Webhook/API，预留 AI 能力接入点 |
| **可观测** | 指标、链路追踪、结构化日志统一接入，支撑运维与排障 |

## 4. 核心域划分

| 域 | 服务 | 职责 |
|----|------|------|
| **平台基础** | gateway, identity | 接入、认证、多租户、RBAC |
| **策略与编排** | policy | 策略 CRUD、版本、下发、冲突检测 |
| **终端与资产** | device, asset | 设备注册、心跳、软硬件清单 |
| **管控能力** | software, dlp, nac, zerotrust, mdm, remote | 各业务策略执行与事件上报 |
| **治理合规** | compliance, audit | 基线检查、合规评分、审计留存 |
| **智能扩展** | ai (预留) | 异常检测、策略推荐、NL 查询 |

## 5. 技术栈摘要

详见 [02-technology-stack.md](./02-technology-stack.md)。

- **后端**：Java 21 + Spring Boot 3.3，单体 API 服务 + 业务模块分包
- **API 通道**：`/api/admin/v1`（管理端）、`/api/app/v1`（手机 App）、`/api/client/v1`（PC 客户端）
- **存储**：MySQL 8（业务）、Redis（缓存/会话）、ClickHouse（审计/遥测）
- **消息**：NATS JetStream（事件驱动）
- **部署**：Docker + Kubernetes，支持私有化单机与集群

## 6. 分阶段建设路线

详见 [09-roadmap.md](./09-roadmap.md)。

| 阶段 | 目标 | 交付模块 |
|------|------|----------|
| **P0 基础平台** | 可纳管、可审计 | identity, device, asset, audit, api三端, agent |

| **P2 安全纵深** | 入网与数据 | nac, dlp |
| **P3 高级能力** | 零信任与移动 | zerotrust, mdm, remote |
